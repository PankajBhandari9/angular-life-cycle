child-after works!

  • To access content we have to first load it and wait if that is in the dom or not. To find out if this is in dom or not we use ngAfterContentInit hook
  • first load sequence ngOnChanges, ngOnInit, Do check, ngAfterContentInit, ngAfterContentChecked, ngAfterViewInit, ngAfterViewChecked
  • second load sequence ngOnChanges, Do check, ngAfterContentChecked, ngAfterViewChecked

On destory

  • It will call when component is not in the dom.
  • useful to release the memory that we have occupied or unsubscibe from the rxjs observables.
  • In the example there is a timer going on, we have to clean when we are not using this component what we can do is on destory we can clean the interval otherwise Do check, ngAfterContentChecked, ngAfterViewChecked these three will be called unnessary.
  • Also if not clear the interval, timer will continue in background and when we goto different route and comeback to this route another timer will continues with parallelly with old one. If we goback and comeback again then it will continue 3rd time as well.
