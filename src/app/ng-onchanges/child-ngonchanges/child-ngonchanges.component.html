About NgOnChanges and NgDoCheck

  1. It will never called if the child data between parent to child or child to parent is not happening
  2. When changes occure then only NgOnChanges is called
  3. It will be called when before the ngOnInit. Then life cycle will be constructor => NgOnChanges => ngOnInit: this will be for the first time only.
  4. After the first load, if any change happened on to the dom then it will only NgOnChanges.
  5. NgOnChanges is for the child component input properties if changes happend it will refelect that
  6. NgOninit is for component first load and changes is for the input for child component
  7. NgOnChanges will not detect the object or array. Because change detection only checks object identity, not object content. Inserts or removals are therefore not detected.
  8. To check the object or array we have to copy array or object when update or create a new object or array each time when it update happens.
  9. If you want to avoid the new object or array creation then we have another hook called ngDoCheck();
  10. NgDocheck will be called every single time when when there is a change in the object or array that we are transfering to the child component. Eg: If object has an input, then whenever there is keypress it will be calling child. Better to prefer create new object or array everytime so that we have previous and current value access.
