NgOninit Vs Constructor

NgOninit
  1. NgOninit Called only once
  2. It is used for the initilization of values like API call in edit and view
  3. It is part of angular, so handled by angular
  4. Binding to the UI is done, so can access the databound property.
Constructor
  1. Constructor is also called once but we don't have to use it for the any work in it.
  2. It is used for dependency injection only.
  3. It is part of Typescript so that we can do a object oriented programming. So it is part of javascript.
  4. Binding didn't happended, only class variable are accessable.
